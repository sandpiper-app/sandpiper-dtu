## Tech Debt Summary

### Phase 02: Shopify Twin Core
- `InventoryItem` GraphQL type and `inventory_items` DB table exist but are unreachable — no `QueryRoot` field, no mutation returns it, no StateManager CRUD methods. Structurally present from schema planning but functionally disconnected. SHOP-01 still satisfied per verifier judgment (covers resources "Sandpiper uses").

### Phase 03: Webhooks & Conformance
- Slack live conformance has no CI schedule job. `conformance-live` only covers Shopify (weekly cron). Slack has `conformance:live` script and `SlackLiveAdapter` ready but no corresponding scheduled job in `conformance.yml`. Slack twin-mode conformance (21 tests) runs on every push/PR.

### Phase 06: Twin UIs
- `@dtu/ui` missing from `tsconfig.base.json` path aliases and twin `tsconfig.json` references. Resolved at runtime via pnpm workspace symlinks, but `tsc --build` incremental compilation cannot track `@dtu/ui` source changes.

### Phase 07: Integration
- Dockerfile `EXPOSE 3000` stale for Slack twin — Slack listens on port 3001 via PORT env var. Functionally harmless (Docker Compose correctly maps 3001).
- Dockerfile build comment (line 30) still references deleted `@dtu/core` package.
- ROADMAP.md Phase 7 entry still shows `[ ]` unchecked and progress table says "0/2 Not started" despite Phase 7 being verified as PASSED.
