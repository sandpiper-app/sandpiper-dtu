    **Test 5 ‚Äî callback exchanges code for a Session (begin‚Üícallback flow):**
    CRITICAL: The callback function requires:
    1. A signed state cookie (set by `begin()`) ‚Äî throws `CookieNotFound` without it
    2. HMAC validation on query params ‚Äî `validateHmac(config)(query)` checks the `hmac` query param
    3. State comparison ‚Äî `safeCompare(query.state, stateFromCookie)` must match

    The correct approach is to call `begin()` first to get the state cookie, then construct a callback request with the matching state, shop, code, and a valid HMAC.

    ```typescript
    it('callback completes OAuth via begin‚Üícallback flow', async () => {
      // Step 1: Call begin() to get the signed state cookie
      const capturedHeaders: Record<string, string | string[]> = {};
      const mockBeginReq = {
        method: 'GET',
        url: 'https://test-app.example.com/auth?shop=dev.myshopify.com',
        headers: { host: 'test-app.example.com' },
      };
      const mockBeginRes = {
        statusCode: 200,
        statusText: 'OK',
        headers: {} as Record<string, string | string[]>,
        setHeader(name: string, value: string | string[]) { capturedHeaders[name.toLowerCase()] = value; this.headers[name] = value; },
        getHeader(name: string) { return this.headers[name]; },
        removeHeader(_name: string) {},
        writeHead(_status: number) {},
        end() {},
        headersSent: false,
      };

      await shopify.auth.begin({
        shop: 'dev.myshopify.com',
        callbackPath: '/auth/callback',
        isOnline: false,
        rawRequest: mockBeginReq as any,
        rawResponse: mockBeginRes as any,
      });

      // Extract Location redirect URL to get the state nonce
      const location = capturedHeaders['location'] as string;
      const redirectUrl = new URL(location);
      const state = redirectUrl.searchParams.get('state')!;

      // Extract the Set-Cookie header(s) for the state cookie
      const setCookieHeaders = capturedHeaders['set-cookie'] ?? [];
      const setCookieArr = Array.isArray(setCookieHeaders) ? setCookieHeaders : [setCookieHeaders];
      // Find the shopify_app_state cookie
      const stateCookie = setCookieArr.find(c => c.includes('shopify_app_state'));

      // Step 2: Build callback request with valid HMAC, state, shop, code
      const callbackParams: Record<string, string> = {
        shop: 'dev.myshopify.com',
        code: 'test-code-123',
        state,
        timestamp: String(Math.floor(Date.now() / 1000)),
        host: Buffer.from('dev.myshopify.com/admin').toString('base64'),
      };
      // Compute HMAC: sort params alphabetically, join as key=value with &
      const sortedEntries = Object.entries(callbackParams).sort(([a], [b]) => a.localeCompare(b));
      const message = sortedEntries.map(([k, v]) => `${k}=${v}`).join('&');
      const hmac = computeShopifyHmac(shopify.config.apiSecretKey, message);
      callbackParams.hmac = hmac;

      const queryString = new URLSearchParams(callbackParams).toString();
      const mockCallbackReq = {
        method: 'GET',
        url: `https://test-app.example.com/auth/callback?${queryString}`,
        headers: {
          host: 'test-app.example.com',
          cookie: stateCookie ? stateCookie.split(';')[0] : '',
        },
      };
      const mockCallbackRes = {
        statusCode: 200,
        statusText: 'OK',
        headers: {} as Record<string, string | string[]>,
        setHeader(name: string, value: string | string[]) { this.headers[name] = value; },
        getHeader(name: string) { return this.headers[name]; },
        removeHeader(_name: string) {},
        writeHead(_status: number) {},
        end() {},
        headersSent: false,
      };

      const { session } = await shopify.auth.callback({
        rawRequest: mockCallbackReq as any,
        rawResponse: mockCallbackRes as any,
      });

      expect(session.shop).toBe('dev.myshopify.com');
      expect(session.accessToken).toBeDefined();
      expect(typeof session.accessToken).toBe('string');
    });
    ```

    PITFALL: The `Cookies` class used by the SDK reads cookies from the `cookie` header and sets them via `Set-Cookie`. The state cookie is signed with `config.apiSecretKey` using `keygrip`. The exact cookie format must match ‚Äî extract the full `shopify_app_state=...` value from Set-Cookie and pass it in the callback request's `cookie` header.

    PITFALL: If the Cookies class requires a specific keygrip signature format that doesn't round-trip through simple header extraction, this test may need adjustment. The executor should check if the cookie value includes a `.sig` companion cookie and include both in the callback request's cookie header.

    FALLBACK: If the begin‚Üícallback flow proves too complex due to cookie signing internals, the executor may test callback indirectly by verifying the callback path raises appropriate errors for invalid inputs (missing state, bad HMAC). This still validates that the SDK's callback validation logic runs correctly against the twin.
