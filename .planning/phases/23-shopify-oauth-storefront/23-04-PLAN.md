<objective>

`X-Shopify-Storefront-Access-Token` while keeping the existing
`Shopify-Storefront-Private-Token` path intact for the pinned SDK.

admin-token rejection, but it only authenticates the private-token header. The
requirement and roadmap success criteria explicitly call out the public header, so this
plan adds that compatibility path without regressing the current private-token behavior.

Output:
- dual-header Storefront token resolution in `graphql.ts`
- focused SHOP-19 coverage for `X-Shopify-Storefront-Access-Token`
- regression protection proving the private-token SDK path still works
</objective>
