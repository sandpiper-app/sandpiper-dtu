<objective>
Create a separate Storefront GraphQL schema and Yoga instance, enforce token-type
auth on the Storefront endpoint so admin tokens are rejected.

Purpose: SHOP-19 requires that admin access tokens are explicitly rejected on the
Storefront endpoint, and that admin-only mutations do not appear in the Storefront
schema. Currently both problems exist — same schema, no token-type check.

Output:
- storefront.graphql SDL (products, shop — no mutations)
- Second Yoga instance in graphql.ts for the Storefront route
- Storefront auth rejects tokens with tokenType 'admin'
- Storefront test updated: seeds a storefront-typed token, adds products query test
  and schema introspection test
</objective>
