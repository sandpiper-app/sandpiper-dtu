<objective>
Fix the `better-sqlite3` ABI mismatch that crashes `pnpm test:sdk` and causes "no test files found".

Purpose: The prebuilt `better-sqlite3` binary was compiled for Node 23 (NMV 137) but the developer's environment runs Node 25 (NMV 141). The fix aligns CI, Docker, and local tooling to Node 22 LTS and adds a `pnpm rebuild better-sqlite3 --build-from-source` step in CI so the native module is always compiled for the running Node version — making the test suite version-agnostic going forward.

Output: `.nvmrc` pinned to 22, all CI jobs in both `conformance.yml` and `e2e.yml` updated from `node-version: 20` to `node-version: 22`, `pnpm rebuild better-sqlite3 --build-from-source` added to the sdk-verification CI job, Dockerfile updated from `node:20-slim` to `node:22-slim`.
</objective>
