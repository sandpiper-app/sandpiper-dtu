- Extends `../../tsconfig.base.json`
- CompilerOptions: composite true, rootDir src, outDir dist
- References: `@dtu/types`, `@dtu/state`, `@dtu/core`

**twins/shopify/src/index.ts:**
- Import Fastify, randomUUID, StateManager
- Export `buildApp(options)` function
- Fastify config: Pino logger with pino-pretty transport, X-Request-Id header, genReqId with randomUUID
- Initialize StateManager with dbPath from env (default ':memory:')
- Decorate Fastify with `stateManager`
- Register health plugin (reuse pattern from example twin)
- Register oauth plugin
- Register admin plugin
- onClose hook to call stateManager.close()
- If run directly (check isMainModule), start server on PORT env (default 3000)

**twins/shopify/src/plugins/oauth.ts:**
- FastifyPluginAsync export as `oauthPlugin`
- POST `/admin/oauth/access_token` endpoint
- Body: `{ code: string }` (simplified - no client_id/client_secret validation for twin)
- Generate token: `crypto.randomUUID()`
- Call `stateManager.createToken(token, 'twin.myshopify.com', 'read_orders,write_orders,read_products,write_products')`
- Return: `{ access_token: token, scope: scopes }`

**twins/shopify/src/plugins/admin.ts:**
- FastifyPluginAsync export as `adminPlugin`
- POST `/admin/reset` - calls `stateManager.reset()`, returns `{ reset: true, timestamp: Date.now() }`
- POST `/admin/fixtures/load` - body: `{ orders?: any[], products?: any[], customers?: any[] }` - iterates and calls createOrder/createProduct/createCustomer, returns `{ loaded: { orders: N, products: M, customers: K } }`
- GET `/admin/state` - returns `{ orders: stateManager.listOrders().length, products: stateManager.listProducts().length, customers: stateManager.listCustomers().length, tokens: (count from listTokens or raw query), webhooks: stateManager.listWebhookSubscriptions().length }`

**twins/shopify/src/services/token-validator.ts:**
- Export `async function validateAccessToken(token: string, stateManager: StateManager): Promise<{ valid: boolean; shopDomain?: string }>`
- Query `stateManager.getToken(token)`
- Return `{ valid: true, shopDomain: tokenRecord.shop_domain }` if found, else `{ valid: false }`
- No caching in Phase 2 (simple approach)

**twins/shopify/src/db/schema.sql:**
- SQL schema matching StateManager tables (for documentation/reference)
- Include all 8 tables with comments

Follow Shopify API versioning convention: endpoints should use `/admin/api/2024-01/` prefix where appropriate (OAuth is `/admin/oauth/access_token` per Shopify docs).
