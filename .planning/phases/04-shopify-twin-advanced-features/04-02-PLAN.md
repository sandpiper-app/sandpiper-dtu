- Change `listProductsStmt` from `ORDER BY created_at DESC` to `ORDER BY id ASC`
- Change `listCustomersStmt` from `ORDER BY created_at DESC` to `ORDER BY id ASC`
- Note: This is a necessary breaking change for deterministic pagination. The `id` column is AUTOINCREMENT, guaranteeing monotonically increasing unique values.

Update `twins/shopify/src/schema/resolvers.ts`:
- Import `encodeCursor`, `decodeCursor` from `../services/cursor.js`
- Create a generic `paginate<T extends { id: number }>(items: T[], args: { first?: number; after?: string; last?: number; before?: string }, resourceType: string)` helper function that:
  1. Sorts items by `id ASC` (should already be sorted from DB, but defensive sort)
  2. If `after`: decode cursor (validate resourceType), filter items where `id > afterId`
  3. If `before`: decode cursor (validate resourceType), filter items where `id < beforeId`
  4. Track `totalAfterCursors = filtered.length` before slicing
  5. If `first`: slice first N items
  6. If `last`: slice last N items
  7. Map to edges with cursor: `{ node: item, cursor: encodeCursor(resourceType, item.id) }`
  8. Build pageInfo: `{ hasNextPage: first != null && totalAfterCursors > first, hasPreviousPage: after != null || (last != null && totalAfterCursors > last), startCursor: edges[0]?.cursor ?? null, endCursor: edges[edges.length-1]?.cursor ?? null }`

- Update `QueryRoot.orders` resolver to use `paginate(orders, { first, after, last, before }, 'Order')`
- Update `QueryRoot.products` resolver to use `paginate(products, { first, after, last, before }, 'Product')`
- Update `QueryRoot.customers` resolver to use `paginate(customers, { first, after, last, before }, 'Customer')`
- Update `Order.lineItems` type resolver to also include `pageInfo` and `cursor` on edges (even though lineItems are JSON-stored, maintain consistent connection structure)
