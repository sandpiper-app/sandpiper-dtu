<objective>
Clean up the broader Shopify harness so future phases inherit version-aware defaults instead of a
hardcoded `2024-01` assumption.

Purpose: Runtime routing alone is not enough. The conformance layer, smoke tests, and in-process
integration tests must stop teaching the rest of the project that Shopify versioning is fixed. This
plan consolidates those assumptions behind a shared helper and adds transport-level assertions for
dual-version routing and echoed response headers.

Output:
1. Shared conformance version helper for Shopify
2. Conformance adapters/suites updated to consume the shared default path/version and honor GraphQL `op.path`
3. Smoke/integration tests extended for `2024-01` plus `2025-01` header parity
</objective>

scattered literals and make future version changes one edit instead of many.
