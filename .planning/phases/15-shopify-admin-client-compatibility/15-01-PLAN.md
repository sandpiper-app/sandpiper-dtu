Key behavioral fact from research (verified from source):
- getHeaders() merge direction: `{...customHeaders, ...config.headers}` — config wins on conflict.
  Callers CANNOT override X-Shopify-Access-Token via getHeaders(). Only non-conflicting custom headers survive.
- getApiUrl(version?) returns the full versioned URL with /admin/api/{version}/graphql.json shape.
  The storeDomain in the URL is dev.myshopify.com (the client-side value, not the rewritten twin URL).
- fetch() returns a raw Response (not parsed). Call response.json() to get the body.
- request() returns ClientResponse<T> with .data and optional .errors.
