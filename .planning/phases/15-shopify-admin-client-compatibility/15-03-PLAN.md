Retry behavior from research:
- REST client retries on RETRIABLE_STATUS_CODES = [429, 500, 503]
- Retry-After header is parsed as parseInt(retryAfter, 10) → '0' parses to 0ms delay
- Pass retries: N per-call (NOT client-level) to be explicit
- retries: 0 via ?? operator does NOT fall back to client default — use retries: 1 for retry test

searchParams encoding from research:
- { limit: 1, fields: 'id,title' } → ?limit=1&fields=id%2Ctitle
- The REST client appends URLSearchParams — pass an object and verify response.url or response.ok
