    1. **Token Validator** (`src/services/token-validator.ts`):
    - `extractBearerToken(request: FastifyRequest): string | null` â€” extracts token from `Authorization: Bearer {token}` header. Returns null if missing or malformed.
    - CRITICAL: Slack Web API returns HTTP 200 with `{ ok: false, error: 'not_authed' }` on auth failures â€” NOT HTTP 401. Only rate limiting uses non-200 status (429).

    2. **Block Kit Validator** (`src/services/block-kit-validator.ts`):
    - `validateBlocks(blocks: unknown[], surface?: 'message' | 'modal'): { valid: boolean; error?: string }`
    - Message surface: max 50 blocks. Modal surface: max 100 blocks. Default to 'message'.
    - Validate each block has a `type` property (string).
    - Valid block types for messages: section, divider, image, actions, context, header, rich_text, video, file. Accept unknown types without error (forward compatibility).
    - Return `{ valid: false, error: 'invalid_blocks' }` if count exceeds limit.
    - Return `{ valid: false, error: 'invalid_blocks_format' }` if blocks is not an array or any block lacks type.

    3. **Rate Limiter** (`src/services/rate-limiter.ts`):
    - `SlackRateLimiter` class with sliding window algorithm (NOT leaky bucket â€” Slack uses simple per-minute windows, unlike Shopify).
    - Constructor accepts optional `tierOverrides` for testing.
    - Built-in tier config (from research):
      - chat.postMessage: 'special', 60/min
      - chat.update: tier 3, 50/min
      - conversations.list: tier 2, 20/min
      - conversations.info: tier 3, 50/min
      - conversations.history: tier 3, 50/min
      - users.list: tier 2, 20/min
      - users.info: tier 4, 100/min
    - `check(method: string, token: string): { retryAfter: number } | null` â€” returns null if allowed, `{ retryAfter: seconds }` if limited.
    - Uses `Map<string, { count: number; resetAt: number }>` keyed by `${method}:${token}`.
    - If current time >= resetAt, start new window. If count >= limit, return retryAfter.
    - `reset(): void` â€” clears all windows.

    Create three Fastify plugin files under `src/plugins/web-api/`, registered from index.ts.

    **Every Web API route handler follows this pattern** (from research):
    1. Extract Bearer token â†’ if missing, return `{ ok: false, error: 'not_authed' }` (HTTP 200)
    2. Validate token against slackStateManager.getToken() â†’ if invalid, return `{ ok: false, error: 'invalid_auth' }` (HTTP 200)
    3. Check rate limit â†’ if limited, return HTTP 429 with `Retry-After` header and body `{ ok: false, error: 'ratelimited' }`
    4. Check error simulation â†’ if configured for this method, return configured error
    5. Process request and return `{ ok: true, ... }` (HTTP 200)

    **CRITICAL anti-pattern:** Slack returns HTTP 200 for ALL Web API errors except rate limits (429). Do NOT return 401, 403, 404 status codes from Web API methods.

    1. **Chat plugin** (`src/plugins/web-api/chat.ts`):

    `POST /api/chat.postMessage`:
    - Body: `{ channel: string, text?: string, blocks?: any[] }` (channel and at least one of text/blocks required)
    - If blocks present, validate via block-kit-validator. Return `{ ok: false, error: 'invalid_blocks' }` on failure.
    - Verify channel exists â†’ if not, return `{ ok: false, error: 'channel_not_found' }`.
    - Generate message ts via generateMessageTs().
    - Store via slackStateManager.createMessage({ channel_id: channel, user_id: botUserIdFromToken, text, blocks: JSON.stringify(blocks), ts }).
    - Return: `{ ok: true, channel, ts, message: { text, ts, type: 'message', user: userId } }`.
    - IMPORTANT: ts is a STRING, not a number (Pitfall 2 from research).

    `POST /api/chat.update`:
    - Body: `{ channel: string, ts: string, text?: string, blocks?: any[] }`
    - If blocks present, validate via block-kit-validator.
    - Verify message exists via slackStateManager.getMessage(ts) â†’ if not, return `{ ok: false, error: 'message_not_found' }`.
    - Update via slackStateManager.updateMessage(ts, { text, blocks }).
    - Return: `{ ok: true, channel, ts, text }`.

    2. **Conversations plugin** (`src/plugins/web-api/conversations.ts`):

    `POST /api/conversations.list` (Slack uses POST even for reads):
    - Body: `{ limit?: number, cursor?: string, types?: string }` (defaults: limit=100)
    - Query slackStateManager.listChannels() with optional cursor-based pagination.
    - Cursor is base64-encoded channel ID for next page.
    - Return: `{ ok: true, channels: [...], response_metadata: { next_cursor: "..." } }`.
    - Each channel object: `{ id, name, is_channel, is_private, is_archived, topic: { value, creator, last_set }, purpose: { value, creator, last_set }, num_members, created }`.

    `POST /api/conversations.info`:
    - Body: `{ channel: string }`
    - Query slackStateManager.getChannel(channel) â†’ if not found, return `{ ok: false, error: 'channel_not_found' }`.
    - Return: `{ ok: true, channel: { ...channelObject } }`.

    `POST /api/conversations.history`:
    - Body: `{ channel: string, limit?: number, cursor?: string, oldest?: string, latest?: string }`
    - Verify channel exists â†’ if not, return `{ ok: false, error: 'channel_not_found' }`.
    - Query slackStateManager.listMessages(channel, { limit: limit ?? 100, cursor, oldest, latest }).
    - Messages ordered newest first (descending ts).
    - Return: `{ ok: true, messages: [...], has_more: boolean, response_metadata: { next_cursor: "..." } }`.
    - Each message: `{ type: 'message', user, text, ts, blocks?, thread_ts?, edited? }`.

    3. **Users plugin** (`src/plugins/web-api/users.ts`):

    `POST /api/users.list`:
    - Body: `{ limit?: number, cursor?: string }`
    - Query slackStateManager.listUsers() with cursor pagination.
    - Return: `{ ok: true, members: [...], response_metadata: { next_cursor: "..." } }`.
    - Each member: `{ id, team_id, name, real_name, profile: { display_name, email }, is_admin, is_bot, deleted }`.

    `POST /api/users.info`:
    - Body: `{ user: string }`
    - Query slackStateManager.getUser(user) â†’ if not found, return `{ ok: false, error: 'user_not_found' }`.
    - Return: `{ ok: true, user: { ...memberObject } }`.

    4. **Register plugins in index.ts**: Import and register chatPlugin, conversationsPlugin, usersPlugin after existing plugin registrations.

    5. **Slack also accepts GET with query params for some methods**, but POST with JSON body is the standard. Implement POST-only for now (all Slack SDK clients use POST).
