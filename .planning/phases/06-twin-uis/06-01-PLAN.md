  <action>
    Create `packages/ui/` as a new workspace package:

    **package.json:**
    - name: `@dtu/ui`, version: `0.1.0`, type: `module`
    - main: `./dist/index.js`, types: `./dist/index.d.ts`
    - dependencies: `eta` (^3.x), `@fastify/view` (^11.x), `@fastify/static` (^8.x), `@fastify/formbody` (^8.x)
    - devDependencies: `fastify` (^5.x), `typescript` (^5.7.3)
    - scripts: `build: tsc --build`
    - Include a `files` field that includes `dist/`, `src/partials/`, and `src/public/` so partials and static assets are available when the package is consumed

    **tsconfig.json:**
    - Extend from root tsconfig pattern (see other packages for reference)
    - Include `src/**/*.ts`, outDir `dist`, composite true, declaration true
    - Do NOT try to compile `.eta` files — they are consumed at runtime via file paths

    **pnpm-workspace.yaml:**
    - Verify `packages/*` glob already covers `packages/ui` (it should since it matches `packages/*`)

    **src/index.ts:**
    - Export `registerUI` async function that accepts Fastify instance and options `{ viewsDir: string, twin: string }`:
      - Registers `@fastify/formbody`
      - Creates Eta instance with `views` pointing to the twin's viewsDir, and shared partials available via `include()` using absolute paths
      - Registers `@fastify/view` with the Eta engine
      - Registers `@fastify/static` with root pointing to `packages/ui/src/public/` and prefix `/ui/static/`, with `decorateReply: false`
    - Export `getPartialsDir()` returning absolute path to `src/partials/`
    - Export `getPublicDir()` returning absolute path to `src/public/`
    - Use `import.meta.dirname` (or `import.meta.url` with `fileURLToPath`) for path resolution

    **src/helpers.ts:**
    - Export `formatDate(unixTimestamp: number): string` — formats Unix timestamp to readable date using `Intl.DateTimeFormat`
    - Export `formatJson(obj: any): string` — `JSON.stringify(obj, null, 2)` with HTML escaping
    - Export `truncate(str: string, maxLen: number): string` — truncates with ellipsis
    - Export `escapeHtml(str: string): string` — escapes &, <, >, ", ' for safe HTML rendering
    - These helpers will be passed to templates as template data by twin UI routes

    **src/partials/layout.eta:**
    - Full HTML5 document structure
    - `<head>`: Pico CSS from CDN (`https://cdn.jsdelivr.net/npm/@picocss/pico@2/css/pico.min.css`), custom styles from `/ui/static/styles.css`, HTMX from CDN (`https://unpkg.com/htmx.org@2.0.4`)
    - `<html data-twin="<%= it.twin %>">` attribute for CSS accent color targeting
    - `<body>` with a container div using CSS grid: sidebar (fixed 220px) + main content area
    - Sidebar region renders `<%~ include('sidebar', it) %>` partial
    - Main content region renders `<%~ it.body %>` (raw HTML content passed by the page template)
    - Flash message area at top of main content using `<%~ include('flash', it) %>`
    - Title tag: `<%= it.pageTitle %> | <%= it.twinName %> Twin`

    **src/partials/sidebar.eta:**
    - `<nav>` element with twin name at top (e.g., "Shopify Twin" or "Slack Twin") styled with accent color
    - Renders `it.navItems` array: each item has `{ label, href, icon?, active }` — renders as `<a>` links
    - Active item highlighted with left border using `var(--twin-accent)` CSS variable
    - Horizontal rule separator before admin section
    - Admin links rendered from `it.adminItems` array: Reset State, Webhooks/Events log

    **src/partials/table.eta:**
    - `<table role="grid">` (Pico CSS uses role="grid" for responsive tables)
    - `<thead>` row from `it.columns` array: each column has `{ key, label }`
    - `<tbody>` rows from `it.rows` array: each row renders cells for each column key
    - Actions column with View link (`<a href="<%= it.basePath %>/<%= row[it.idKey] %>">`) and Edit link
    - Delete button using HTMX: `hx-delete="<%= it.basePath %>/<%= row[it.idKey] %>"`, `hx-target="closest tr"`, `hx-swap="delete"`
    - Empty state: if `it.rows.length === 0`, show "No items" message
    - Row count displayed above table: `<small><%= it.rows.length %> item(s)</small>`

    **src/partials/detail.eta:**
    - Heading with entity name/title
    - Field list: iterates `it.fields` array, each `{ label, value }` rendered as `<dt>`/`<dd>` in a description list
    - Action buttons: Edit link, Delete button (HTMX with redirect after delete), Back to list link
    - Raw JSON toggle using `<details><summary>Raw JSON</summary><pre><code><%= it.rawJson %></code></pre></details>` (per user decision)

    **src/partials/form.eta:**
    - `<form method="POST" action="<%= it.action %>">`
    - Iterates `it.fields` array: each `{ name, label, type, value?, required?, placeholder?, options? }`
    - Renders `<input>` for text/number/email, `<select>` for select type, `<textarea>` for textarea type
    - Submit button with `it.submitLabel` text (default "Save")
    - Cancel link back to `it.cancelHref`
    - HTML5 validation attributes: `required` when field.required is true

    **src/partials/flash.eta:**
    - Conditionally renders `it.flash` object if present
    - `<div role="alert">` with success (green border) or error (red border) styling
    - Message text from `it.flash.message`

    **src/public/styles.css:**
    - CSS custom properties for twin accent colors:
      ```css
      :root { --twin-accent: #666; }
      [data-twin="shopify"] { --twin-accent: #008060; }
      [data-twin="slack"] { --twin-accent: #611f69; }
      ```
    - Body layout: CSS grid with sidebar + main, `min-height: 100vh`
    - Sidebar: fixed width 220px, background slightly darker than white, full height, padding
    - Sidebar twin name: accent color, font-weight bold
    - Active nav link: left border with accent color
    - Table tweaks: compact padding for dense display (user decision: dense data tables, many rows visible)
    - Delete button styling: small, secondary, no border
    - Details/summary for JSON toggle: monospace font, overflow scroll
    - No dark mode — light theme only (user decision)
    - No animations or transitions — utilitarian (user decision)
  </action>
