    **package.json:** Add `@dtu/ui: workspace:*` to dependencies.

    **src/plugins/ui.ts:** Create a Fastify plugin (use `fastify-plugin` to ensure access to stateManager decorator from parent scope) that:

    1. Calls `registerUI(fastify, { viewsDir: path.join(import.meta.dirname, '../views'), twin: 'shopify' })` from @dtu/ui
    2. Imports helpers from `@dtu/ui` (formatDate, formatJson, truncate)

    **Navigation config** (passed to all page renders):
    ```typescript
    const navItems = [
      { label: 'Orders', href: '/ui/orders' },
      { label: 'Products', href: '/ui/products' },
      { label: 'Customers', href: '/ui/customers' },
    ];
    const adminItems = [
      { label: 'Admin', href: '/ui/admin' },
      { label: 'Webhooks', href: '/ui/admin/webhooks' },
    ];
    ```

    **Helper function** for common render data:
    ```typescript
    function pageData(nav: string, pageTitle: string, extra: Record<string, any> = {}) {
      return { twin: 'shopify', twinName: 'Shopify', navItems, adminItems, nav, pageTitle, ...extra };
    }
    ```

    **Routes — Orders:**
    - `GET /ui` → redirect to `/ui/orders` (default landing page)
    - `GET /ui/orders` → list all orders from `stateManager.listOrders()`. Render `orders/list.eta` with columns: ID, Name, Total Price, Financial Status, Fulfillment Status, Created At. Pass helpers.
    - `GET /ui/orders/:id` → get order by ID from `stateManager.getOrder(parseInt(id))`. If not found → 404. Render `orders/detail.eta` with all fields + raw JSON.
    - `GET /ui/orders/new` → render `orders/form.eta` with empty fields for create mode.
    - `POST /ui/orders` → parse form body. Generate GID using `createGID('Order', Date.now() + Math.floor(Math.random() * 100000))`. Call `stateManager.createOrder({ gid, name, total_price, currency_code })`. Trigger webhooks: find subscriptions for `orders/create` topic, enqueue each. Redirect to `/ui/orders` (PRG pattern).
    - `GET /ui/orders/:id/edit` → get order, render `orders/form.eta` with populated fields for edit mode.
    - `POST /ui/orders/:id` → parse form body. Call `stateManager.updateOrder(parseInt(id), { name, total_price, currency_code })`. Trigger webhooks for `orders/update`. Redirect to `/ui/orders/:id`.
    - `DELETE /ui/orders/:id` → call `stateManager.database.prepare('DELETE FROM orders WHERE id = ?').run(parseInt(id))`. Trigger webhooks for `orders/delete`. Return empty string with 200 (HTMX will remove the row).

    **Routes — Products:**
    - `GET /ui/products` → list all products. Columns: ID, Title, Vendor, Product Type, Created At.
    - `GET /ui/products/:id` → detail view with raw JSON.
    - `GET /ui/products/new` → create form. Fields: title, description, vendor, product_type.
    - `POST /ui/products` → create product with generated GID. Trigger `products/create` webhooks. Redirect.
    - `GET /ui/products/:id/edit` → edit form with populated fields.
    - `POST /ui/products/:id` → update product via `stateManager.updateProduct()`. Trigger `products/update` webhooks. Redirect.
    - `DELETE /ui/products/:id` → delete from products table. Return empty for HTMX.

    **Routes — Customers:**
    - `GET /ui/customers` → list all customers. Columns: ID, Email, First Name, Last Name, Created At.
    - `GET /ui/customers/:id` → detail view with raw JSON.
    - `GET /ui/customers/new` → create form. Fields: email, first_name, last_name.
    - `POST /ui/customers` → create customer with generated GID. Redirect.
    - `GET /ui/customers/:id/edit` → edit form with populated fields.
    - `POST /ui/customers/:id` → update customer. Note: StateManager may not have an updateCustomer method — add a direct SQL update if needed. Redirect.
    - `DELETE /ui/customers/:id` → delete from customers table. Return empty for HTMX.

    **Routes — Admin:**
    - `GET /ui/admin` → render admin dashboard showing state counts (orders, products, customers, tokens, webhooks). Include "Reset State" button that POSTs to `/ui/admin/reset`.
    - `POST /ui/admin/reset` → call existing reset logic (stateManager.reset(), rateLimiter.reset()). Redirect to `/ui/admin`.
    - `GET /ui/admin/webhooks` → list webhook subscriptions from `stateManager.listWebhookSubscriptions()`. Show topic and callback_url.

    **Webhook dispatch pattern** (reuse in all create/update handlers):
    ```typescript
    async function dispatchWebhooks(fastify: FastifyInstance, topic: string, payload: any) {
      const subs = fastify.stateManager.listWebhookSubscriptions()
        .filter((s: any) => s.topic === topic);
      for (const sub of subs) {
        await fastify.webhookQueue.enqueue({
          id: randomUUID(),
          topic,
          callbackUrl: sub.callback_url,
          payload,
          secret: fastify.webhookSecret,
        });
      }
    }
    ```

    **IMPORTANT:** Register the `/ui/orders/new` route BEFORE `/ui/orders/:id` so that `new` is not captured as an `:id` parameter. Same for products and customers.

    **src/index.ts:** Import and register the UI plugin:
    ```typescript

    // ... after existing plugin registrations:
    await fastify.register(uiPlugin);
    ```
