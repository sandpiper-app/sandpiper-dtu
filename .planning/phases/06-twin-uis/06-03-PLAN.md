  <action>
    **package.json:** Add `@dtu/ui: workspace:*` to dependencies.

    **src/plugins/ui.ts:** Create a Fastify plugin (use `fastify-plugin` for parent scope access) that:

    1. Calls `registerUI(fastify, { viewsDir: path.join(import.meta.dirname, '../views'), twin: 'slack' })` from @dtu/ui
    2. Imports helpers from `@dtu/ui`, ID generators from `../services/id-generator.js`

    **Navigation config:**
    ```typescript
    const navItems = [
      { label: 'Channels', href: '/ui/channels' },
      { label: 'Users', href: '/ui/users' },
    ];
    const adminItems = [
      { label: 'Admin', href: '/ui/admin' },
      { label: 'Events', href: '/ui/admin/events' },
    ];
    ```

    **Routes — Channels:**
    - `GET /ui` → redirect to `/ui/channels`
    - `GET /ui/channels` → list channels from `slackStateManager.listChannels()`. Columns: ID, Name, Is Private, Topic, Num Members, Created.
    - `GET /ui/channels/new` → create form. Fields: name (text, required), is_private (checkbox), topic (text), purpose (text).
    - `GET /ui/channels/:id` → channel detail page. This is the MESSAGE TIMELINE view — the key differentiator from Shopify. Show channel info at top, then list messages in chronological order below. Messages from `slackStateManager.listMessages(id)`. Each message shows: user name (looked up from slackStateManager.getUser()), text, timestamp, blocks (if present). Include a "Post Message" form at the bottom of the page (inline, not a separate page — this matches the Slack UX metaphor of typing in a channel). Also show raw JSON toggle for the channel object.
    - `POST /ui/channels` → create channel with `generateChannelId()`. Call `slackStateManager.createChannel({ id, name, is_private, topic, purpose, creator: 'U_BOT_TWIN' })`. Dispatch event using `eventDispatcher` (channel_created event) if event subscriptions exist. Redirect to `/ui/channels`.
    - `POST /ui/channels/:id/message` → post a message in this channel. Parse form body for `text` and optional `user_id` (default to `U_BOT_TWIN`). Call `slackStateManager.createMessage({ ts: generateMessageTs(), channel_id: id, user_id, text })`. Dispatch message event using `eventDispatcher.dispatchMessageEvent()` or equivalent. Redirect to `/ui/channels/:id` (back to the channel detail/timeline).
    - `GET /ui/channels/:id/edit` → edit form for channel name, topic, purpose.
    - `POST /ui/channels/:id` → update channel via `slackStateManager.updateChannel(id, { name, topic, purpose })`. Redirect to `/ui/channels/:id`.
    - `DELETE /ui/channels/:id` → delete channel: `slackStateManager.database.prepare('DELETE FROM slack_channels WHERE id = ?').run(id)`. Return empty for HTMX.

    **Routes — Users:**
    - `GET /ui/users` → list users from `slackStateManager.listUsers()`. Columns: ID, Name, Real Name, Email, Is Admin, Is Bot.
    - `GET /ui/users/new` → create form. Fields: name (text, required), real_name (text), display_name (text), email (email), is_admin (checkbox), is_bot (checkbox).
    - `GET /ui/users/:id` → detail view with all user fields + raw JSON toggle.
    - `POST /ui/users` → create user with `generateUserId()`. Call `slackStateManager.createUser({ id, name, real_name, display_name, email, is_admin, is_bot, team_id: 'T_TWIN' })`. Redirect.
    - `GET /ui/users/:id/edit` → edit form with populated fields.
    - `POST /ui/users/:id` → update user. SlackStateManager may not have updateUser — add direct SQL if needed: `UPDATE slack_users SET name=?, real_name=?, display_name=?, email=? WHERE id=?`. Redirect.
    - `DELETE /ui/users/:id` → delete from slack_users table. Return empty for HTMX.

    **Routes — Admin:**
    - `GET /ui/admin` → state counts: channels, users, messages, tokens, event_subscriptions. "Reset State" button.
    - `POST /ui/admin/reset` → call `slackStateManager.reset()`, `rateLimiter.reset()`. Redirect to `/ui/admin`.
    - `GET /ui/admin/events` → list event subscriptions from database: `slackStateManager.database.prepare('SELECT * FROM slack_event_subscriptions').all()`. Show app_id, request_url, event_types.

    **IMPORTANT route ordering:** Register `/ui/channels/new` and `/ui/users/new` BEFORE `/:id` routes.

    **src/index.ts:** Import and register the UI plugin:
    ```typescript
    import uiPlugin from './plugins/ui.js';
    // ... after existing plugin registrations:
    await fastify.register(uiPlugin);
    ```

    **Template files:**

    **Channels list (channels/list.eta):**
    - Table with columns: ID, Name, Private, Topic, Members
    - "New Channel" button

    **Channel detail (channels/detail.eta):**
    - Channel info section: name, topic, purpose, created date
    - Message timeline section: chronological list of messages
    - Each message: `<div class="message">` with user name, text, timestamp
    - Messages displayed with monospace timestamp, user bold, text normal (Slack-like compact)
    - If blocks present on a message, show "[has blocks]" indicator
    - "Post Message" form at bottom: text input + optional user_id dropdown + submit button
    - Raw JSON toggle for channel object
    - Back to channels link

    **Channel form (channels/form.eta):**
    - Fields: name, is_private (checkbox), topic, purpose

    **Users list (users/list.eta):**
    - Table: ID, Name, Real Name, Email, Admin, Bot
    - "New User" button

    **User detail (users/detail.eta):**
    - Fields: Name, Real Name, Display Name, Email, Is Admin, Is Bot, Team ID, Timezone
    - Raw JSON toggle

    **User form (users/form.eta):**
    - Fields: name (required), real_name, display_name, email, is_admin (checkbox), is_bot (checkbox)

    **Admin index (admin/index.eta):**
    - State counts: channels, users, messages, tokens, event subscriptions
    - Reset button
    - Load fixtures note

    **Admin events (admin/events.eta):**
    - Table of event subscriptions: App ID, Request URL, Event Types
    - Empty state if none
