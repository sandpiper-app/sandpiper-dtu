Create `docker-compose.twin.yml` at the repository root that defines both twin services on a shared Docker network.

**shopify-twin service:**
- Build from root context using Dockerfile with `TWIN_NAME: shopify` build arg
- Port mapping: `${SHOPIFY_TWIN_PORT:-3000}:3000` (configurable host port, internal always 3000)
- Environment: PORT=3000, DB_PATH=:memory:, NODE_ENV=production
- Health check: `["CMD", "node", "healthcheck.mjs"]` with interval=10s, timeout=3s, start_period=5s, retries=3
- Network: twin-network

**slack-twin service:**
- Build from root context using Dockerfile with `TWIN_NAME: slack` build arg
- Port mapping: `${SLACK_TWIN_PORT:-3001}:3001` (configurable host port, internal always 3001)
- Environment: PORT=3001, DB_PATH=:memory:, NODE_ENV=production
- Health check: `["CMD", "node", "healthcheck.mjs"]` with interval=10s, timeout=3s, start_period=5s, retries=3
- Network: twin-network

**Network:** `twin-network` with bridge driver. This allows Sandpiper (when added to the same network) to reach twins by service name: `http://shopify-twin:3000` and `http://slack-twin:3001`.

<task type="auto">
  <name>Task 2: Create GitHub Actions E2E workflow</name>
  <files>.github/workflows/e2e.yml</files>
  <action>
Create `.github/workflows/e2e.yml` — a GitHub Actions workflow that builds twin Docker images, starts the stack, runs integration tests, and tears down.

**Trigger:** on push to main and on pull_request (same as conformance.yml pattern).

**Job: e2e-twins**
- runs-on: ubuntu-latest
- Steps:
  1. `actions/checkout@v4`
  2. Build and start twin stack: `docker compose -f docker-compose.twin.yml up -d --build --wait` with timeout-minutes: 5
  3. Verify twin health: `curl -sf http://localhost:3000/health | jq .` and `curl -sf http://localhost:3001/health | jq .`
  4. Set up pnpm: `pnpm/action-setup@v4` with version 9 (matches project)
  5. Set up Node.js: `actions/setup-node@v4` with node-version 20 and cache pnpm
  6. Install dependencies: `pnpm install`
  7. Build packages: `pnpm build` (needed for vitest to resolve @dtu/* packages)
  8. Run integration smoke tests: `pnpm exec vitest run tests/integration/smoke.test.ts` with env vars:
     - `SHOPIFY_API_URL: http://localhost:3000`
     - `SLACK_API_URL: http://localhost:3001`
  9. Run full test suite (optional additional confidence): `pnpm test` — runs all unit and conformance tests
  10. Tear down: `docker compose -f docker-compose.twin.yml down --volumes` with `if: always()`

Key considerations:
- This workflow requires NO sandbox API credentials (INTG-02 success criteria)
- The `--wait` flag on docker compose up blocks until health checks pass
- The smoke tests from Plan 01 validate twins respond correctly
- Follow the same CI patterns as the existing conformance.yml workflow
- Do NOT duplicate the conformance tests — they run in their own workflow
  </action>
