  <action>
    Keep `METHOD_SCOPES` unchanged for catalog and seeding purposes, but add dedicated helper exports below `checkScope()` for conversation readers.

    Add these concrete helpers:
    1. `resolveConversationTypeScopes(method: 'conversations.list' | 'conversations.info' | 'conversations.history', types: string[]): string[]`
      - Map `public_channel` -> `channels:read` or `channels:history`
      - Map `private_channel` -> `groups:read` or `groups:history`
      - Map `im` -> `im:read` or `im:history`
      - Map `mpim` -> `mpim:read` or `mpim:history`
      - Deduplicate while preserving the order above
    2. `resolveChannelClass(channel: { id: string; is_channel: number | boolean; is_private: number | boolean }): 'public_channel' | 'private_channel' | 'im' | 'mpim'`
      - `channel.id.startsWith('D_')` => `im`
      - `!channel.is_channel && channel.is_private` => `mpim`
      - `channel.is_private` => `private_channel`
      - otherwise => `public_channel`
    3. `checkResolvedScopes(requiredScopes: string[], tokenScope: string): ScopeCheckResult | null`
      - Same behavior as `checkScope`, but works from a resolved array instead of a method key

    Do not change `allScopesString()` or the existing `METHOD_SCOPES['conversations.*']` entries. Existing broad-seeded tokens must keep working.
  </action>
