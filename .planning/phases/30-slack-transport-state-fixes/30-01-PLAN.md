<objective>
Fix Slack event delivery to use direct `fetch()` instead of routing through `WebhookQueue`, so delivered events carry only Slack signature headers (`X-Slack-Signature`, `X-Slack-Request-Timestamp`) rather than Shopify-specific headers (`X-Shopify-Hmac-Sha256`, `X-Shopify-Topic`, `X-Shopify-Webhook-Id`).

Purpose: The `WebhookQueue` → `deliverWebhook()` path unconditionally injects Shopify headers before merging `delivery.headers` — the Shopify headers always win because they are set first and `delivery.headers` is spread on top. Bypassing the queue entirely with direct `fetch()` (the same pattern already used in `interactions.ts`) is the cleanest fix.

Output: Updated `event-dispatcher.ts` with `WebhookQueue` dependency removed; `pnpm test:sdk` SLCK-16a test GREEN.
</objective>
