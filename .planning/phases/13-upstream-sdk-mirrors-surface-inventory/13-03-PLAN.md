  <instructions>
    Create repo-owned forks of the three upstream SDK monorepos on GitHub. Use the same GitHub account or organization that owns this repo so the submodule URLs are accessible in CI without extra auth.

    Fork these three repos:
    1. https://github.com/Shopify/shopify-app-js  →  your fork URL (e.g., https://github.com/YOUR_ORG/shopify-app-js.git)
    2. https://github.com/slackapi/node-slack-sdk  →  your fork URL (e.g., https://github.com/YOUR_ORG/node-slack-sdk.git)
    3. https://github.com/slackapi/bolt-js  →  your fork URL (e.g., https://github.com/YOUR_ORG/bolt-js.git)

    To fork: Visit each URL above, click "Fork", select your account/org, keep the default name, click "Create fork".

    After forking, you need the exact git SHA that corresponds to each installed package version. Use these version-to-tag mappings to find the SHA in each fork:

    For shopify-app-js (contains both @shopify/admin-api-client and @shopify/shopify-api):
    - Look for a release tag `@shopify/shopify-api@12.3.0` in the fork's tag list
    - Run in your fork: `git ls-remote origin "refs/tags/@shopify/shopify-api@12.3.0"` to get the commit SHA
    - If the tag points to a tag object (annotated): dereference it with `^{}` suffix

    For node-slack-sdk (contains @slack/web-api and @slack/oauth):
    - Look for release tags `@slack/web-api@7.14.1` and `@slack/oauth@3.0.4`
    - The monorepo may use a single commit for coordinated releases — one SHA can cover both packages
    - Use the @slack/web-api@7.14.1 tag SHA as the node-slack-sdk submodule pin

    For bolt-js (root package is @slack/bolt):
    - Look for release tag `@slack/bolt@4.6.0`
    - Use that tag's SHA as the bolt-js submodule pin

    Record the three fork URLs and three SHAs. You will provide them when prompted.
  </instructions>
