    Require:

    Require:
    - `grant_type === 'client_credentials'`
    - `client_id`

    Do not require `code`.

    3. `refresh_token` branch.
    Require:
    - `grant_type === 'refresh_token'`
    - `client_id`
    - `client_secret`
    - non-empty `refresh_token`
    Presence-only is sufficient for Phase 39; do not add refresh-token storage semantics.

    4. Token-exchange branch.
    Require:
    - `grant_type === 'urn:ietf:params:oauth:grant-type:token-exchange'`
    - `client_id`
    - `client_secret`
    - non-empty `subject_token`
    - `subject_token_type === 'urn:ietf:params:oauth:token-type:id_token'`
    - `requested_token_type` equal to exactly one of:
      - `urn:shopify:params:oauth:token-type:online-access-token`
      - `urn:shopify:params:oauth:token-type:offline-access-token`

    On missing fields, return `400` with `{ error: 'invalid_request', error_description: '...' }` where the description names the missing field set for that branch.
    On unsupported `requested_token_type`, return:
    - `error: 'invalid_request'`
    - `error_description: 'requested_token_type must be urn:shopify:params:oauth:token-type:online-access-token or urn:shopify:params:oauth:token-type:offline-access-token'`

    Preserve:

    - current `ADMIN_SCOPES`
