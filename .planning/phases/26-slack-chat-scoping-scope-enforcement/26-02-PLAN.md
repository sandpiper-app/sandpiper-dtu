    Add the following export to the bottom of twins/slack/src/services/method-scopes.ts (after allScopesString):

    ```typescript
    export interface ScopeCheckResult {
      error: 'missing_scope';
      needed: string;
      provided: string;
    }

    /**
     * Check whether a token's scope string satisfies the required scopes for a method.
     *
     * Returns null if all required scopes are present or if the method has no requirements.
     * Returns a ScopeCheckResult if any required scope is missing.
     *
     * Used by Phase 26 scope enforcement in all Slack twin plugins.
     */
    export function checkScope(method: string, tokenScope: string): ScopeCheckResult | null {
      const required = METHOD_SCOPES[method];
      if (!required || required.length === 0) return null;

      const granted = new Set(tokenScope.split(',').map(s => s.trim()).filter(Boolean));
      const missing = required.find(s => !granted.has(s));
      if (!missing) return null;

      return { error: 'missing_scope', needed: missing, provided: tokenScope };
    }
    ```

    Do not modify any existing exports. The existing METHOD_SCOPES and allScopesString() are unchanged.

    ### 2. Update checkAuthRateError return type and body

    Change the function signature from `Promise<string | null>` to `Promise<{ token: string; tokenRecord: any } | null>`.

    After the existing `tokenRecord` null check, add scope enforcement + headers:
    ```typescript
    // SLCK-18: scope enforcement
    const scopeCheck = checkScope(method, tokenRecord.scope);
    if (scopeCheck) {
      await reply.status(200).send({ ok: false, ...scopeCheck });
      return null;
    }

    // SLCK-19: pre-set scope response headers (set before rate limit / error sim
    // so they accompany whatever response is eventually sent from the handler)
    const accepted = METHOD_SCOPES[method]?.join(',') ?? '';
    reply.header('X-OAuth-Scopes', tokenRecord.scope);
    reply.header('X-Accepted-OAuth-Scopes', accepted);
    ```

    Change the final `return token;` to `return { token, tokenRecord };`.

    ### 3. Update all call sites of checkAuthRateError in chat.ts

    Every existing call `const token = await checkAuthRateError(...)` becomes:
    ```typescript
    const authResult = await checkAuthRateError(fastify, request, reply, 'method.name');
    if (!authResult) return;
    const { token, tokenRecord } = authResult;
    ```
    Apply this pattern to ALL handlers that call checkAuthRateError:
    - chat.postMessage, chat.delete, chat.postEphemeral, chat.getPermalink, chat.meMessage,

    For handlers that already call checkAuthRateError and then use `token` downstream (e.g., chat.meMessage uses tokenRecord.user_id), the destructuring makes tokenRecord available without a second getToken call.
